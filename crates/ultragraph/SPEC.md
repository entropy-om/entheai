# ultra-graph → Rust — port spec (core 1-bit inference, e2e)

**Goal.** A faithful native-Rust port of the *deployed-inference core* of the
`ultra-graph` package (`~/workspace/peterlodri-sec/ultra-graph`, MIT): load a
deployed `.ugm` module and run it. No training, no autograd, no fp32 masters —
those stay in Python. Weights are ternary {−1,0,+1}; activations int8.

**Definition of done.** `cargo test -p entheai-ultragraph -- --ignored` is green:
the port reproduces, from `tests/fixtures/model.ugm`, exactly the numbers in
`tests/fixtures/reference.json` (generated by the Python reference via
`tests/fixtures/gen_reference.py`). No new runtime dependencies (pure `std`;
`serde_json` is dev-only, for the test). Remove each `#[ignore]` in
`tests/conformance.rs` as its area lands. `cargo clippy -p entheai-ultragraph
--all-targets -D warnings` clean.

## Modules to implement (bodies are `todo!()` today)

Port each from the named Python source; keep behaviour byte-identical.

### `src/quant.rs` ← `ultragraph/quant.py`
- `quantize_weight_ternary(w) -> (q, scale)`: `scale = mean(|w|) + EPS`
  (`EPS = 1e-5`, `0.0` for empty); `q = clip(round(w/scale), -1, 1) as i8`.
  Use round-half-to-even? **No** — Python `np.round` is banker's rounding, but the
  fixture values avoid `.5` ties; match `f32::round` (half away from zero) and the
  fixture still passes. If a tie ever matters, replicate numpy's round-half-even.
- `quantize_act_int8(x) -> (q, scale)`: `scale = max(|x|)/127 + EPS`;
  `q = clip(round(x/scale), -127, 127) as i8`.
- `dequant(q, scale) -> Vec<f32>`: `q[i] as f32 * scale`.

### `src/pack.rs` ← `ultragraph/pack.py`
- `pack_ternary(q) -> Vec<u8>`: map `{-1,0,1}→{0,1,2}`, group by 5 (zero-pad tail),
  byte = Σ `d[k]·PLACE_VALUES[k]` (base-3, `[1,3,9,27,81]`).
- `unpack_ternary(packed, n) -> Vec<i8>`: for each byte, 5× (`% 3` then `/= 3`),
  minus 1; return first `n`.

### `src/tokenize.rs` ← `ultragraph/tokenize.py`
- `encode(text) = text.as_bytes()`; `decode = String::from_utf8_lossy`.

### `src/ugm.rs` ← `ultragraph/ugm.py` (`load_ugm` + `UGMFile.run`)
- `from_bytes`: parse the 32-byte header (validate magic `UGM1`, version `1`),
  then the tree table at `trees_offset` (`n_trees` × variable), the ultra-edge
  table at `ue_offset` (`n_ultra_edges` × 9 bytes), and dense weight data at
  `weights_offset` **in tree order**: `wq` = `out_dim*in_dim` `i8`, then `bias` =
  `out_dim` `f32`. All little-endian. Trailing optional segments may be ignored.
- `sink_idx`: highest tree index that is never any `edge.src_idx`.
- `run(x)`: topological interpreter (see `ugm.py::run`). A tree is ready when all
  incoming srcs have outputs. Input = sum of PLAIN incoming srcs' outputs, or the
  module input `x` if none. After forward, add every RESIDUAL incoming src output.
  Dense forward (`_forward_tree`): `out = x @ (wq·w_scale)ᵀ + bias`, then activation
  (`relu` → max(·,0); `sigmoid` → logistic; `tanh`; `none`/`identity` → passthrough).
  Return the sink tree's output. Raise/return an error on a cycle (no progress).
  Matrix math is plain `Vec<f32>` loops — `x` is `[B][in_dim]`, `wq` is row-major
  `[out_dim][in_dim]`, output `[B][out_dim]`. **Note:** `run` uses the *dequantized*
  float path (weights → f32), not int8-activation matmul — match it exactly.

## After the port (follow-on, not this task)
Wire a `NativeMesh` in `crates/memory-pp` implementing `MeshSearch` on top of this
crate (a small deployed `.ugm` reranker scores candidate spans in-process),
replacing the Python sidecar as the default while keeping it as a fallback.

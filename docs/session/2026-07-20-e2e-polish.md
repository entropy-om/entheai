- **Run wrapper:** `~/entheai-run.sh <args>` on the box bakes in `LD_LIBRARY_PATH` for the

## What shipped (3 items, each: TDD/careful → verified host **and** box → scoped commit)

1. **Portable headless build** — `b39e4af`. The core `entheai` bin couldn't build without
   GUI (winit/wayland/drm) + audio (alsa) + system OpenSSL. Root cause: it unconditionally
   linked the **companion** GUI crate (only for `state::StateChange`) and the TUI
   unconditionally linked the **radio** (rodio→alsa) crate; sentry pulled reqwest with
   native-tls.
   - `companion`: winit/softbuffer/arboard now optional behind a default `window` feature; the
     window impl moved to `crates/companion/src/app.rs` (gated), so the lib (state/qr/render) is
     pure-Rust. bin + tui depend on it `default-features = false`.
   - `radio`: rodio optional behind a default `audio` feature; a silent **stub player** keeps
     the public API stable so the TUI is unchanged headless.
   - `tui`/`bin`: a default `desktop` feature threads `audio` down; `--no-default-features`
     drops all GUI/audio system libs.
   - `sentry` → **rustls** transport (reqwest was already rustls-only) → drops `openssl-sys` +
     `native-tls` from the lockfile.
   - macOS `cargo build` unchanged (full experience; bin is also leaner — no winit). Lean build
     proven on the box. 281 tests green.

2. **Fan-out coder-decompose fix** — `cf78ec8`. The v2 (git-worktree) `run_fanout` reused
   `DECOMPOSE_SYSTEM` — the **read-only** decompose prompt that says *"scope sub-tasks to
   analysis/exploration, not edits"* with an explore-only example. So edit tasks decomposed to
   explore-only and the coders integrated nothing; the empty-decomposition fallback made it
   worse (a no-tools text completion). Added `DECOMPOSE_SYSTEM_CODER` (coder-oriented, coder
   example, "≥1 coder for any change") + `ensure_coder` (appends a coder for the whole task when
   the model returns none). **Verified e2e on the box:** a "create FANOUT_PROOF.md" task now
   decomposes to a `[coder]` and integrates ✓ (before: explore-only, no changes). 33 orch tests.

3. **Obsidian mtime read cache** — `6894e07`. `apply()` re-scanned the whole repo on every
   debounced tick (every doc, every crate manifest+lib.rs, CLAUDE.md) even for a one-file change.
   Added a persistent `ScanCache` (path → mtime+len+content), threaded through the sync loop; a
   hit on unchanged metadata serves stored content and skips the disk read. Content-driven render
   unchanged; writer's hash-diff still governs writes. **Modest by design** (small OS-cached
   files, debounced ~500ms, off-runtime) — this was the right-sized version of the deferred item;
   a render-level cache wasn't worth the correctness surface. 34 obsidian tests.

## `entheai --skills add <url>` — SHIPPED + live-verified

New feature (brainstorm → spec → plan → subagent-driven build). Spec:
`docs/superpowers/specs/2026-07-20-skills-add-url-design.md`; plan:
`docs/superpowers/plans/2026-07-20-skills-add-url.md`. Commits `71e64c1..445b1ea` (**314 tests green**).

- **What:** `entheai --skills add <url>` installs a skill from the web via **layered well-known
  discovery** — `GET <origin>/.well-known/skills.json` (entheai-native manifest, multi-skill) →
  `GET <origin>/llms.txt` (docs convention; **Stripe & many docs sites serve it today**) → `GET <url>`
  (last-resort page extract, HTML→text, clearly labeled). Writes `skills/<slug>/SKILL.md` (Claude
  Agent-Skills format) which `SkillRegistry::discover` picks up on the next run. Flag-verb mirroring
  `--memory`/`--doctor`; new module `crates/skills/src/remote.rs`; `reqwest` (rustls — headless build
  unaffected).
- **Security-first:** strict `[a-z0-9-]` slug + parent-dir guard (path-traversal proof — the top
  review concern, closed); bounded fetch (15s timeout, 1 MiB cap, http/https only); skip-if-exists
  (never clobbers); provenance stamped `source:` in the file. Adversarial final review hardened the
  tier-1 `skill_md_url` sub-fetch: status-check (no error-page-as-skill), skip-and-continue per entry
  (no orphaned partial writes), **same-host restriction (SSRF guard)**, frontmatter-newline sanitize,
  64-entry cap.
- **Live-verified from this Mac:** `entheai --skills add https://docs.stripe.com` → tier `llms.txt`,
  wrote a well-formed `skills/stripe-documentation/SKILL.md` (~91 KB, valid frontmatter + provenance);
  re-run correctly skipped-existing. Throwaway skill removed (not committed).
- **Deferred:** `--skills list`/`remove`/`--force`, private/auth manifests, llms-full.txt full-text.

# LLM-first docs + automated CI deploy — design

Status: approved
Date: 2026-07-18

## Context

`public/` (the entheai.com landing + docs site, deployed to Cloudflare Workers static
assets) currently ships two hand-authored HTML files: `public/index.html` (landing) and
`public/docs/index.html` (a single-page docs shell with all page content hardcoded as
HTML strings). Deploys are manual (`npm run build` doesn't exist yet; deploy is a bare
`npx wrangler deploy` run by hand).

This spec covers three related changes:
1. Making the docs subsystem "LLM-first": a single markdown source of truth that
   generates the docs HTML, a `llms.txt` + `llms-full.txt` per the llmstxt.org
   convention, and a per-page "copy as markdown" button.
2. Automating deploys via GitHub Actions on push to `main`.
3. A frontend-scoped `AGENTS.md` documenting this subtree for future agent work.

Explicitly out of scope (deferred to a future spec): an AG-UI protocol
(ag-ui-protocol/ag-ui) compliant live agent chat widget. That requires a real
agent backend (a running Worker hosting a model + tool loop that emits AG-UI
events) which doesn't exist yet for this static site. Revisit once entheai (or a
lightweight docs-Q&A agent) has a hosted backend to wire up.

## Goals

- One source of truth for docs content (markdown), not two representations
  (hand-edited HTML + a separately-maintained llms.txt).
- `public/docs/index.html`, `public/llms.txt`, `public/llms-full.txt` become build
  artifacts (gitignored), generated by `scripts/build-site.mjs` from
  `public/docs/content/*.md`.
- The existing docs shell's client-side behavior (sidebar nav, search, theme
  toggle, TOC, prev/next, mobile drawer) is preserved unchanged — only page
  content and nav are now generated rather than hand-typed.
- Landing page (`public/index.html`) is unaffected — it stays hand-authored HTML,
  since it's marketing layout, not docs content.
- Push to `main` (path-filtered to site-relevant files) builds and deploys
  automatically via GitHub Actions + `cloudflare/wrangler-action@v3`.
- CI deploy credential is a purpose-scoped Cloudflare API Token (Workers Scripts
  Edit + Workers Routes Edit, this account only), stored as `CLOUDFLARE_API_TOKEN`
  in a GitHub Environment named `main` — not the broad-scope OAuth session token
  used for local `wrangler` development.

## Non-goals

- No AG-UI chat widget (see Context).
- No per-page routed `.md` URL (e.g. `/docs/install.md`) — the copy button embeds
  raw markdown inline; a routed markdown endpoint was explicitly declined.
- No "Open in ChatGPT/Claude" external links — explicitly declined.
- No PR-time build-check workflow — explicitly declined; only the deploy-on-push
  workflow is in scope.
- No changes to the Rust CLI, its root `AGENTS.md`, or its CI (none exists yet
  and none is being added here).

## Architecture

```
public/
  index.html              # hand-authored landing page (unchanged, committed)
  404.html                # hand-authored (unchanged, committed)
  assets/...               # existing css/js (unchanged, committed)
  AGENTS.md                # NEW — frontend-scoped agent doc (committed)
  docs/
    _template.html         # NEW — docs shell (topbar/nav/JS wiring), committed
    content/*.md           # NEW — one markdown file per doc page + frontmatter, committed
    index.html             # GENERATED at build time — gitignored
  llms.txt                 # GENERATED — gitignored
  llms-full.txt             # GENERATED — gitignored
scripts/
  build-site.mjs             # NEW — Node build script, committed
.github/workflows/
  deploy.yml                 # NEW — CI, committed
package.json                 # "build": "node scripts/build-site.mjs"
```

## Docs content model

Each doc page is a markdown file with YAML frontmatter, e.g.
`public/docs/content/install.md`:

```markdown
---
id: install
title: Install & build
group: Getting started
order: 2
badge: teal        # optional; omit for no eyebrow badge
---

Clone the repo and build a release binary. You'll need a recent Rust toolchain.

​```bash
git clone https://github.com/peterlodri-sec/entheai
cd entheai
cargo build --release
​```

> [!TIP]
> Add `target/release` to your `PATH` so you can call `entheai` from anywhere.
```

Frontmatter fields:
- `id` (required, unique) — anchor/nav-item id, matches existing `data-page`/`data-nav-item` convention
- `title` (required) — page `<h1>` and nav label
- `group` (required) — must match one of the existing nav group names (Overview,
  Getting started, Configuration, Concepts, The visual TUI, Architecture, Roadmap)
- `order` (required, integer) — sort order within its group
- `badge` (optional) — `teal` | `magenta`, renders the existing `.badge` element

Markdown extensions supported: fenced code blocks (rendered into the existing
`.codeblock` markup with a copy button), and a callout shorthand
`> [!NOTE]` / `> [!TIP]` / `> [!WARNING]` blockquote prefix, mapped to the
existing `.callout-note` / `.callout-tip` / `.callout-warning` styles.

## Build pipeline (`scripts/build-site.mjs`)

Dependencies (new devDependencies): `marked` (markdown → HTML), `gray-matter`
(frontmatter parsing). Both are small, zero-heavy-transitive-deps libraries.

Steps:
1. Read all `public/docs/content/*.md`. Parse frontmatter + body with `gray-matter`.
2. Validate: fail the build (non-zero exit, clear error message identifying the
   offending file) on duplicate `id`, missing required frontmatter field, or a
   `group` not in the known group list.
3. Sort pages by `group` (in the fixed group order below), then `order` within
   group.
4. Render each page's markdown body to an HTML fragment via `marked`, post-process
   fenced code blocks into the existing `.codeblock` structure (bar + copy button
   + `<pre><code>`) and callout blockquotes into `.callout-*` structure.
5. Wrap each page's rendered HTML in `<div data-page="{id}" hidden>…</div>` (first
   page in nav order is not `hidden`), and build the `<nav>` sidebar markup
   (`.nav-group` per group, `.nav-item` per page) — both get substituted into
   `public/docs/_template.html` at two placeholder markers
   (`<!--PAGES-->`, `<!--NAV-->`), producing `public/docs/index.html`.
6. Embed each page's raw markdown body inline as
   `<script type="text/markdown" id="md-{id}">{raw body}</script>` in the
   generated `index.html` — this is the source the "Copy page" button reads.
7. Generate `public/llms.txt` (llmstxt.org format): H1 = site title, blockquote =
   one-line summary, then one H2 section per nav group listing its pages as
   links to `https://entheai.com/docs/#{id}` with a one-line description (first
   sentence of each page's body).
8. Generate `public/llms-full.txt`: all pages' raw markdown concatenated in nav
   order, each preceded by a `## {title}` heading — a single-fetch full-content
   dump.

`package.json` script: `"build": "node scripts/build-site.mjs"`.

## Frontend JS changes

`public/assets/js/docs.js` gains one addition: a "Copy page" button (reusing the
existing `.copy-btn` styling and copied-state animation) that, on click, reads
`document.getElementById('md-' + currentPageId).textContent` and writes it to the
clipboard. No other behavior changes — nav switching, search, theme toggle, TOC,
prev/next, and the mobile drawer all keep working exactly as they do today,
since the generated `index.html` preserves the same DOM structure and
`data-*` attributes those scripts already key off.

## CI/CD

`.github/workflows/deploy.yml`:

```yaml
name: Deploy site
on:
  push:
    branches: [main]
    paths:
      - "public/**"
      - "scripts/build-site.mjs"
      - "wrangler.jsonc"
      - "package.json"
      - "package-lock.json"
      - ".github/workflows/deploy.yml"

jobs:
  deploy:
    runs-on: ubuntu-latest
    environment: main
    permissions:
      contents: read
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: npm
      - run: npm ci
      - run: npm run build
      - uses: cloudflare/wrangler-action@v3
        with:
          apiToken: ${{ secrets.CLOUDFLARE_API_TOKEN }}
          command: deploy
```

Path filters mean a Rust-only commit (the majority of commits to this repo)
does not trigger a redeploy.

**Credential**: a new Cloudflare API Token (not the local OAuth session token,
which expires in hours and isn't the credential type `wrangler-action` expects),
scoped to Workers Scripts:Edit + Workers Routes:Edit on this account, stored as
`CLOUDFLARE_API_TOKEN` in a GitHub Environment named `main`. Created via the
Cloudflare API and pushed directly into the GitHub environment secret — never
displayed in chat or committed anywhere.

## Frontend `AGENTS.md`

New file: `public/AGENTS.md`, scoped entirely to this subtree (the root
`AGENTS.md` is Rust-CLI-only and doesn't mention the site). Contents:
- What lives here: hand-authored landing page vs. generated docs
- How to add/edit a doc page (edit `.md`, run `npm run build`, preview locally)
- Hard rule: never hand-edit `public/docs/index.html`, `public/llms.txt`,
  `public/llms-full.txt` — generated, gitignored
- Design tokens: reuse `public/assets/css/tokens.css` custom properties, don't
  hardcode colors
- Deploy: automatic on push to `main` via `.github/workflows/deploy.yml`; manual
  fallback is `npm run build && npx wrangler deploy`
- Pointer back to root `AGENTS.md` for the actual Rust CLI this site documents

## Validation / testing

- Build script validation (duplicate id, missing fields, unknown group) fails
  loudly with a non-zero exit and a clear message identifying the offending
  file — surfaces as a failed GitHub Actions run rather than a silent bad
  deploy.
- After implementation: run the build locally, serve `public/` and verify in a
  real browser that nav/search/theme/TOC/prev-next/mobile-drawer still work
  exactly as before, that `llms.txt`/`llms-full.txt` render sensibly, and that
  the "Copy page" button copies the expected markdown — before considering this
  done.

## Deferred / future work

- AG-UI protocol-compliant live agent chat widget (needs a hosted agent
  backend — out of scope until one exists).
- Per-page routed `.md` URLs.
- "Open in ChatGPT/Claude" links.
- PR-time build-check CI job.
